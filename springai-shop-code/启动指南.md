# 商城系统启动指南

## 前置准备

### 1. 安装必要软件
- JDK 17 或更高版本
- Node.js 18 或更高版本
- MySQL 8.0 或更高版本
- Redis 6.0 或更高版本
- Maven 3.6 或更高版本

### 2. 初始化数据库

```sql
-- 在MySQL中执行以下命令
mysql -u root -p

-- 然后执行SQL脚本
source D:/project/springai-shop-code/app-backend/src/main/resources/sql/init.sql
```

或者直接在MySQL客户端中打开并执行 `init.sql` 文件。

### 3. 配置后端

编辑 `app-backend/src/main/resources/application.yml`:

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/shop_db?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
    username: root        # 修改为你的MySQL用户名
    password: root        # 修改为你的MySQL密码
  
  data:
    redis:
      host: localhost     # Redis地址
      port: 6379          # Redis端口
      password:           # Redis密码(如果有)
  
  ai:
    openai:
      api-key: your-api-key-here  # 如需使用AI功能,填入OpenAI API Key
```

## 启动步骤

### 方式一: 命令行启动

#### 1. 启动后端

```bash
# 进入后端目录
cd D:\project\springai-shop-code\app-backend

# 使用Maven启动
mvn spring-boot:run

# 或者先编译再运行
mvn clean package
java -jar target/springai-shop-backend-1.0.0.jar
```

后端启动成功后会显示:
```
========================================
   商城系统后端启动成功!
   API地址: http://localhost:8080/api
========================================
```

#### 2. 启动前端

打开新的命令行窗口:

```bash
# 进入前端目录
cd D:\project\springai-shop-code\app-frontend

# 首次运行需要安装依赖
npm install

# 启动开发服务器
npm run dev
```

前端启动成功后会显示:
```
VITE v5.x.x  ready in xxx ms

➜  Local:   http://localhost:3000/
```

### 方式二: IDE启动

#### IntelliJ IDEA

1. **后端**:
   - 打开 `app-backend` 文件夹
   - 等待Maven依赖下载完成
   - 找到 `ShopApplication.java`
   - 右键点击 -> Run 'ShopApplication'

2. **前端**:
   - 打开 `app-frontend` 文件夹
   - 打开Terminal
   - 运行 `npm install`
   - 运行 `npm run dev`

## 常见问题

### 1. 后端启动失败

**问题**: 数据库连接失败
```
解决方案:
- 检查MySQL是否启动
- 检查数据库shop_db是否创建
- 检查application.yml中的数据库配置是否正确
```

**问题**: Redis连接失败
```
解决方案:
- 检查Redis是否启动
- Windows: 运行 redis-server.exe
- Mac/Linux: 运行 redis-server
```

### 2. 前端启动失败

**问题**: npm install 失败
```
解决方案:
- 检查Node.js版本 (需要18+)
- 清除缓存: npm cache clean --force
- 删除node_modules重新安装
```

**问题**: 端口被占用
```
解决方案:
修改 vite.config.js 中的端口号
server: {
  port: 3001,  // 改为其他端口
}
```

### 3. AI功能无法使用

```
解决方案:
- 在 application.yml 中配置有效的 OpenAI API Key
- 或者暂时不使用AI功能,其他功能不受影响
```

### 4. 跨域问题

前端已经配置了代理,正常情况下不会出现跨域问题。
如果遇到问题,检查 `vite.config.js` 中的 proxy 配置。

## 功能测试流程

1. **注册/登录**
   - 访问首页,点击右上角"注册"
   - 填写信息完成注册
   - 使用注册的账号登录

2. **浏览商品**
   - 首页查看推荐商品
   - 点击"全部商品"查看所有商品
   - 可以使用分类筛选和搜索功能

3. **加入购物车**
   - 点击商品进入详情页
   - 选择数量,点击"加入购物车"
   - 或直接点击"立即购买"

4. **下单购买**
   - 点击头部购物车图标
   - 查看购物车商品
   - 点击"结算"
   - 填写收货信息
   - 提交订单

5. **查看订单**
   - 点击用户菜单 -> "我的订单"
   - 可以查看订单状态
   - 待付款订单可以取消
   - 已发货订单可以确认收货

6. **AI客服** (需配置API Key)
   - 首页右下角点击"AI助手"
   - 输入问题与AI对话
   - 可以咨询商品信息、购物问题等

## 开发建议

### 后端开发
- 修改代码后需要重启服务
- 使用 `mvn spring-boot:run` 可以快速重启
- 日志输出在控制台,便于调试

### 前端开发
- 支持热更新,修改代码自动刷新
- 使用浏览器开发者工具调试
- Vue DevTools 可以帮助调试Vue组件
